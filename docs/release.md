# Release Guide

This guide describes the release flow for GitHub Releases and the Tauri updater.

## Release Policy

SayNext uses Semantic Versioning.

- Patch, such as `0.1.1`: bug fixes and copy changes.
- Minor, such as `0.2.0`: new features that keep existing behavior stable.
- Major, such as `1.0.0`: stable public release or breaking changes after 1.0.

## Prepare a Version

Update all app version files:

```bash
npm run version:set -- 0.1.1
```

Update `CHANGELOG.md`, then run:

```bash
npm run check
```

Optional local installer build:

```bash
npm run bundle
```

Updater builds require the signing private key:

```bash
TAURI_SIGNING_PRIVATE_KEY="$(cat /path/to/saynext-updater.key)" \
TAURI_SIGNING_PRIVATE_KEY_PASSWORD="" \
npm --workspace apps/desktop run tauri:build -- --bundles app
```

## Create a GitHub Release

Commit the release changes:

```bash
git add package.json package-lock.json apps/desktop/package.json apps/desktop/src-tauri/tauri.conf.json CHANGELOG.md
git commit -m "Release v0.1.1"
```

Tag and push:

```bash
git tag v0.1.1
git push origin main
git push origin v0.1.1
```

Pushing the tag starts `.github/workflows/release.yml`.

The workflow builds:

- macOS Apple Silicon
- macOS Intel
- Windows x64

The workflow creates a draft GitHub Release. Review the generated installers, updater signatures, and `latest.json`, edit release notes if needed, then publish the draft.

## Current Signing Status

The first open-source releases can be unsigned for OS-level code signing, but users may see operating-system security warnings.

The Tauri updater has its own signing requirement and cannot install unsigned updates.

Before a wider public launch:

- Configure Apple Developer ID signing and notarization for macOS. See [Apple Developer ID Signing](apple-signing.md).
- Configure a Windows code signing certificate.
- Keep the Tauri updater private key backed up securely.

## In-App Updates

In-app updates use the Tauri updater plugin and GitHub Release `latest.json`.

Required GitHub Actions secrets:

- `TAURI_SIGNING_PRIVATE_KEY`: the full private key content generated by `tauri signer generate`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`: empty string for the current key, because it was generated without a password

Important: `0.1.0` cannot update itself because it did not include the updater. Users need to manually install `0.1.1` once. Releases after that can be installed from inside SayNext.
